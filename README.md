# 🩺 PainPoint Tracker (통증 다이어리) - PWA MVP

바쁜 직장인들이 매일 겪는 근골격계 통증과 생활습관을 최소한의 터치로 신속하게 기록하고, 누적된 데이터를 AI가 정밀하게 분석하여 맞춤형 자가진단 피드백(주간/월간 리포트) 및 맞춤 건강 상품(커머스)을 제안하는 **모바일 웹 기반 PWA(Progressive Web App) MVP** 서비스입니다.

---

## 🚀 주요 제공 기능

### 1. 직관적인 통증 및 습관 기록 대시보드 (`/`)
* **신체 대분류 및 정밀 세부 부위 터치**: 전면/후면 전환이 가능한 휴먼 SVG 바디 맵을 통해 아픈 곳(머리, 목, 어깨, 손목, 등, 무릎, 발 등)을 한 번에 터치해 선택하고, 각 부위별 손가락 마디, 관절, 척추 마디 등 정밀한 세부 좌표까지 직관적으로 설정할 수 있습니다.
* **생활습관 및 통증 레벨 슬라이더**: 통증 강도(Lv.1~10), 수면 시간(4~10시간), 자세 및 요추 정렬 지표(1~5점)를 슬라이더 형태로 편리하게 조절합니다.
* **기록 이력 타임라인**: 최근 작성된 기록 5건을 일목요연하게 확인하고, 개별 삭제 등의 제어가 가능합니다.

### 2. AI 기반 정밀 분석 리포트 (`/reports`)
* **데이터 시각화**: 누적된 기록의 평균 통증 지수, 수면 및 자세 점수를 분석하고, 대분류별 통증 빈도 및 세부 부위 비율을 프로그레스 바 형태의 차트로 시각화합니다.
* **실시간 AI 연동 리포팅**:
  * **광고 후 무료 분석**: 시뮬레이션 광고 카드를 보고 난 후 로컬 알고리즘 기반 엔진을 통한 리포트를 즉각 발행합니다.
  * **API 연동 정밀 연산**: OpenAI (`gpt-4o-mini`) 또는 Google Gemini (`gemini-3.5-flash`) API Key를 입력하여 외부 AI 전문가의 맞춤 진단을 실시간 연동합니다.
* **종합 피드백 및 추천 진료과 제공**: AI가 예상 질환/상태(예: 요추 추간판 탈출증, 초기 목 디스크, 수근관 증후군 등)와 맞춤 임상 진료과(정형외과, 신경과, 재활의학과 등)를 예측 및 추천합니다.

### 3. 맞춤형 케어 쇼핑관 (`/commerce`)
* **증상 매칭 상품 추천**: 분석된 리포트의 주요 통증 부위에 따라 맞춤 도구(버티컬 마우스, 경추베개, 허리 쿠션 등) 및 맞춤 영양제(오메가3, 마그네슘 등)를 자동으로 큐레이션합니다.
* **인터랙티브 맵 필터링**: 쇼핑관 내의 3D 신체 지도를 직접 터치하여 특정 관절/부위에 도움을 주는 보조 기구들을 간편하게 필터링할 수 있습니다.
* **최저가 가격 비교**: 네이버 쇼핑 검색 엔진을 경유해 실시간 최저가 정보를 열람하고 외부 구매처로 바로 이동할 수 있습니다.

## 🛠️ 최근 업데이트 내역 (2026-06-15)

### 1. 개발 환경에서의 서비스 워커(PWA) 오작동 및 무한 루프 해결
* **문제 배경**: 로컬 서버 접속(`localhost` 및 `192.168.x.x` 등 사설 IP) 시 서비스 워커(`sw.js`)가 Next.js의 개발용 핫 리로드(HMR) 및 Webpack 요청(`_next/webpack-hmr`)을 가로채고 캐시된 메인 HTML을 잘못 응답하면서 브라우저가 무한히 새로고침되는 오류가 발생했습니다.
* **수정 내용**:
  * `sw.js`와 `layout.tsx`에 로컬 접속 IP 대역을 완벽히 인식하는 감지 로직을 추가하여, 개발 환경에서는 서비스 워커의 `fetch` 가로채기를 완전 바이패스(우회)하고 서비스 워커를 스스로 등록 해제(Unregister)시키도록 변경하여 무한 새로고침 루프를 원천 차단했습니다.
  * 빌드 시 서비스 워커 캐싱 리스트에서 404 에러를 내던 `/globals.css`를 제외하여 터미널 에러 로그를 해결했습니다.

### 2. Google Gemini API 연동 404 오류 수정
* **문제 배경**: 구글 Generative Language API 모델이 구형인 `gemini-1.5-flash`로 하드코딩되어 있어, 해당 모델 단종 이후 API 연동 시 `404 Not Found` 에러가 발생했습니다.
* **수정 내용**:
  * `reportGenerator.ts` 내부의 호출 모델 식별자를 현재 공식 지원하는 최신 안정화 모델인 **`gemini-3.5-flash`**로 마이그레이션하여 실시간 AI 리포트 기능이 완벽히 동작하도록 보완했습니다.

## 💻 실행 및 설치 방법

### 개발 환경 (Development)
의존성 패키지를 설치한 후, Next.js 개발 서버를 구동합니다.

```bash
# 1. 패키지 설치
npm install

# 2. 로컬 개발 서버 실행
npm run dev
```
기본적으로 브라우저에서 `http://localhost:3000` 또는 터미널에 기재되는 사설 IP 주소(`http://192.168.x.x:3000`)로 접속해 테스트하실 수 있습니다.

### 프로덕션 빌드 (Production Build)
최적화된 프로덕션 패키지를 빌드하고 실행합니다.

```bash
# 1. 프로덕션 빌드 컴파일
npm run build

# 2. 프로덕션 서버 시작
npm run start
```

## 📦 기술 스택 및 환경
* **Framework**: Next.js 16.2 (App Router, Turbopack)
* **Language**: TypeScript
* **Styling**: Tailwind CSS
* **Mock DB / Storage**: LocalStorage 기반 Local Database Mockup Layer (`src/lib/supabase.ts`)
* **Service Worker**: PWA 오프라인 모드 자산 캐싱 (`public/sw.js`)
* **AI API**: Google Gemini (`gemini-3.5-flash`), OpenAI (`gpt-4o-mini`)
